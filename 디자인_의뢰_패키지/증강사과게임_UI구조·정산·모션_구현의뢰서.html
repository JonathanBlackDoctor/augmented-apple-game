00한 장 요약 — 무엇을 맡기나

바쁘면 이 절만 읽으세요. 이 의뢰서는 세 개의 산출물 기둥을 요청합니다.

① 구조 재설계 (IA)

화면·상태·플로우의 정보 구조를 다시 짭니다. 흩어진 5개 모드, 중복된 오버레이/HUD, 우회하는 온라인 플로우를 일관된 한 시스템으로.

② 정밀 UI 디자인

모든 화면의 모든 상태(빈·로딩·에러·승/패/신기록 등)를 모바일+데스크톱으로. 설정·정산은 특히 깊게.

③ 모션 구현

드래그·제거·정산·전환·카운트다운·MMR 연출을 구현 가능한 스펙(지속·이징·트리거)으로. CSS/Lottie/Pixi로 바로 이식.

가장 중요한 한 가지

지금 게임은 점수가 조용히 쌓이고 라운드가 조용히 넘어갑니다. 30초의 긴장과 빌드 성장의 쾌감이 연출(정산 모먼트)로 환산되지 못하고 있어요. 이 의뢰의 심장은 "라운드 정산"을 카타르시스의 순간으로 설계·구현하는 것입니다(§07).

현재 코드에 있는 것 문제 구조적 결함 재설계 요청 모션 움직임 스펙 신규 새로 추가

01게임 30초 브리핑 (문서 자체 완결)

디자인 결정을 위한 최소 이해. 4겹으로 쌓여 있습니다.

  • 코어 루프 — 격자에 1~9 숫자 사과가 깔리고(기본 17×10=170칸), 드래그로 만든 직사각형 안 사과들의 합이 정확히 10이면 제거+득점. 제한 시간(기본 30초) 안에 최대 점수.
  • 증강 레이어 — 라운드 사이 3택1·리롤 없음으로 능력을 누적해 "빌드"를 만듦. 티어 상승: 실버(R1·2) → 골드(R3·4) → 프리즘(R5). 계열 6종(시간·콤보·보드·룰·하이리스크·견제).
  • 대결 레이어 — 5라운드×30초, 라운드 점수 누적 + 라운드 승자 보너스 → 최종 1위. 현재 AI 대결 + 온라인 1:1 구현. MMR/ELO + 7티어(아이언~마스터).
  • 소셜 레이어 — 링크/코드 초대, (예정) 데일리 시드 챌린지·공유 카드.

꼭: 직접 손맛 체험

npm install && npm run dev → localhost:5173. 드래그→제거→콤보→증강 선택을 직접 만지셔야 "정산을 어디에 끼워야 하는지" 감이 옵니다. 30초의 리듬이 이 의뢰의 전부입니다.

02기술 구조 — 두 개의 렌더링 세계 + 상태 모델

"어디에 무엇을 납품하면 구현되나"의 지도. 디자인은 이 경계 위에서 결정됩니다.

① React / CSS (DOM)

홈·HUD·모든 오버레이/모달·설정·정산·로비·리더보드·토스트. → CSS 스펙 + 토큰(CSS 변수)으로 납품. src/ui/**

② PixiJS (캔버스)

사과 격자·사과·선택 박스·제거 파티클·점수 팝업·AI 미니뷰·(예정)견제 FX. → 벡터 드로잉 스펙 또는 스프라이트. src/board/BoardView.ts

③ 살아있는 하늘 (배경)

아침→낮→해질녘→밤→황혼 이음매 없는 1일 루프가 게임 시계에 연동(시간 증강·일시정지 시 멈춤). 시그니처 자산. DayNightSky.tsx

문제 토큰이 두 곳에 흩어져 있다

색·치수가 src/ui/styles.css(CSS 변수 215개)와 src/board/theme.ts(Pixi 숫자 토큰)에 이중 관리됩니다. 새 시스템은 하나의 토큰 소스(JSON/Style-Dictionary)에서 DOM·Pixi 양쪽으로 파생되게 정리해 주세요. 사과 룩은 별도 핸드오프 패키지(apple_design_handoff/)가 이미 있으니 그 스펙과 충돌 없이 통합해야 합니다.

상태 모델 (zustand) — 모든 화면 전환의 근원

UI는 두 축의 곱으로 라우팅됩니다(src/app/store.ts, App.tsx):

축값의미
phasehome · round · roundCheck · augment · result매치 내부 진행 단계
modesolo · augment · versus · online · leaderboard어떤 종류의 플레이인가

문제 online·leaderboard는 phase 머신을 우회한다

solo/augment/versus는 phase로 흐르지만, online은 자체 스토어(onlineStore)·자체 HUD·자체 증강 패널·자체 결과 카드를 따로 가집니다. 그 결과 같은 화면이 두 벌 존재하고 서로 어긋나 있습니다(아래 §03·§05.4 참조). 재설계의 1차 목표는 이 분기를 하나의 정산/오버레이 시스템으로 수렴시키는 것입니다.

03정보 구조(IA) & 화면 플로우 — 현재 지도 + 재설계

이 절이 "구조적 재설계"의 뼈대입니다. 먼저 지금 흐름을 그대로 펼칩니다.

현재 플로우 현재

홈 → 빠른 한 판 (solo) →1R·30s round → result
홈 → 5R 증강 (augment) → round →정산 없음 augment 선택 ↺×5 result
홈 → AI 대결 (versus) → round → roundCheck3.5s 자동 → augment12s 자동 ↺×5 result+MMR
홈 → 온라인 (online) → menu/hosting/connecting → countdown → round ↔ augment ↺×5 → result+MMR
홈→랭킹 (leaderboard)→홈

구조적 문제 7가지 문제

#문제근거 (코드 기준)
1홈의 평면적 메뉴CTA 5개(빠른 한 판/5라운드 증강/AI 대결/친구와 1:1/랭킹)가 위계·그룹 없이 일렬. 설정·도움말은 하단 작은 ghost. "혼자/대결/순위"의 묶음이 안 읽힘.
2정산 모먼트의 비대칭솔로 증강 모드는 라운드 정산(점검)이 아예 없음 — 라운드 끝→바로 증강 선택. 대결만 roundCheck 존재. 같은 "라운드를 끝냈다"가 모드마다 다르게 끝남.
3HUD 3벌 분산Hud·VersusHud·OnlineHud가 별개 컴포넌트. 콤보 칩 위치/일시정지 버튼 유무/상대 +N 팝업 유무가 제각각.
4증강 패널 2벌AugmentOverlay(아이콘+타이머) vs OnlineScreen 내부 인라인 패널(계열 아이콘·카운트다운 없음)이 갈라짐.
5결과 카드 3벌ResultOverlay(솔로)·VersusResult·OnlineScreen 결과가 각자 마크업. 정산 연출을 한 번 만들면 셋 다 못 씀.
6빌드(보유 증강) 가시성OwnedAugments 사이드바가 대결에서만 표시(isVersus && …). 솔로 증강 모드도 빌드를 쌓는데 안 보임. 증강 선택 화면에서도 현재 빌드를 못 봄(블라인드 픽).
7오버레이 dismiss 규칙 불일치설정·도움말은 배경 클릭 닫힘, 증강·결과·일시정지·정산은 안 닫힘. 공유 전환/백드롭/포커스 규칙이 없음.

재설계 방향 재설계

  • 홈을 "허브"로 재구성: 1차 행동(이어하기/빠른 한 판) → 모드 그룹(혼자 · 대결 · 순위) → 유틸(설정·도움말·프로필)의 명확한 위계. 모드 카드에 상태 미리보기(최고점·내 티어·온라인 가능여부).
  • 단일 매치 셸: round → 정산 → (증강) → round 루프를 모든 모드 공통으로. 솔로 증강 모드에도 가벼운 라운드 정산을 추가(§07). 온라인을 별도 셸에서 공통 셸로 흡수.
  • 오버레이 시스템 단일화: 백드롭·진입/이탈 전환·포커스 트랩·dismiss 규칙·z-레이어를 §04.1로 통일. 증강/정산/결과/설정/도움말/일시정지가 같은 셸의 변형.
  • HUD 1 시스템 · 3 변형(솔로 / 대결 / 온라인) — §04.2.
  • 네비게이션 일관성: 뒤로/홈/일시정지의 위치와 동작을 전 화면 통일. 모바일 제스처(스와이프-다운으로 정산 스킵 등) 정의.

04전역 시스템 재설계 (파운데이션)

화면별로 들어가기 전, 모든 화면이 공유하는 다섯 가지 시스템을 먼저 확정합니다.

4.1 오버레이 / 모달 시스템 재설계 모션

현재 모든 오버레이는 .overlay > .card 한 겹이고 전환이 없습니다. 통일 사양을 요청합니다:

백드롭일관된 dim + (가벼운) 배경 처리. 보드가 비치는 라운드 중 오버레이와, 보드를 가리는 메뉴 오버레이를 구분.
진입/이탈백드롭 페이드 + 카드 rise(아래에서 14px↑ + scale .98→1). 이탈은 역재생(빠르게). 지속·이징은 §09 표.
dismiss 규칙닫기 가능(설정·도움말·프로필) vs 불가(정산·증강 선택·카운트다운)를 명시적 유형으로. 닫기 가능은 백드롭 클릭+ESC+스와이프다운, 닫기 버튼 항상 존재.
z-레이어하늘(0) → 보드 캔버스(1) → HUD(2) → 빌드 사이드바(2) → 오버레이 백드롭(3) → 오버레이 카드(4) → 토스트(5) → 인앱 배너(6). 충돌 없는 맵 납품.
포커스/접근성모달 오픈 시 포커스 트랩, 첫 포커스 대상 지정, 닫으면 호출 버튼으로 복귀. aria-modal·라벨.
스택일시정지 위에 설정/도움말이 겹쳐 열림(현재 구조). 2단 스택의 백드롭·전환 정의.

4.2 HUD 시스템 재설계

3개 컴포넌트를 1 시스템 · 3 변형 · 공유 슬롯으로. 현재 슬롯 구성(코드 기준):

변형좌중앙우
솔로 Hud라운드칩 R n/5 · 점수 · "점"시간 바 + 남은 초(≤5초 경고)콤보 칩 · 일시정지
대결 VersusHud나: 라벨·점수·콤보R n/5 · 라운드 승수 me:opp · 일시정지 · 시간 바·초상대: 이름·점수·+N 플로팅
온라인 OnlineHud내 이름·점수·콤보R n/5 · 승수 · 시간 바·초(비활성 시 "—")상대 이름((대기))·점수

요청

공통 슬롯(나 / 시간·라운드 / 상대|콤보·일시정지)을 정의하고, 모드별로 슬롯을 채우거나 비웁니다. 일시정지·콤보·상대 +N·저시간 경고는 어느 변형이든 같은 컴포넌트·같은 모션. 세로(모바일)에서 HUD가 보드 상단을 덮지 않도록 세이프 영역·높이 정의.

4.3 디자인 토큰 단일화 재설계

색·간격·라운드·그림자·이징·z·모션 듀레이션을 시맨틱 토큰으로. DOM(CSS 변수)과 Pixi(theme.ts 숫자) 양쪽 파생. 현재 값은 전면 교체 대상이며, 어떤 변수 군이 있는지만 참고:

paper ink apple gold valid(green) prism-a prism-b

반경 18/14px · 그림자 부드러운 갈색 대형 · 이징 cubic-bezier(.22,1,.36,1) · 폰트 Pretendard(한글)+tabular-nums(숫자). 보드 비율: 사과 반지름 0.42·셀, 숫자 0.48·셀, 선택 박스 모서리 0.16·셀.

4.4 레이아웃 · 반응형 골격 재설계

  • 320px(모바일 세로) → 데스크톱 전 구간. 보드는 항상 화면에 꽉 맞춤(정사각 셀), 세로에서는 세로 격자로 스왑(작은 화면 셀 ~20px에서도 1~9 즉독).
  • 세이프 에어리어(viewport-fit=cover 노치), HUD·하단 컨트롤·토스트가 이를 존중.
  • 오버레이 카드 최대폭·내부 그리드(증강 3카드, 설정 행, 정산 점수)의 모바일 재배치(가로→세로) 규칙.

4.5 모션 원칙 모션

듀레이션 스케일

micro 90–140ms(버튼/토글) · UI 180–260ms(오버레이/칩) · 정산/연출 320–900ms(점수 카운트·바·MMR). 입력 피드백은 항상 micro.

레이어드 & 의미

중요도=무게. 긍정(합10·신기록·승리)은 크고 화려하게, 중립(전환)은 절제. 콤보가 길수록 이펙트가 누적 강화.

입력 우선

30초 플레이 중에는 어떤 모션도 입력을 지연/가림 금지. 보드 드래그 미리보기·제거는 60fps 보장(무거운 blur/filter 자제).

감속 선호

prefers-reduced-motion에서 모든 장식 모션 off + 즉시 상태. 정산은 "스킵 가능"으로 동등 정보 제공.

5.1 홈 HomeScreen.tsx

현재

중앙 home-card: SVG 로고("10" 새긴 사과) → 타이틀 "증강 사과게임" → 한 줄 설명 → CTA 5개 세로 나열 → (최고점>0이면) "최고 점수 N" → 힌트 → 하단 ⚙ 설정/? 도움말 작은 버튼. 뒤로 낮밤 하늘 자동 순환(60초).

문제
  • 5개 CTA가 동일 위계 — "지금 한 판"과 "랭킹 보기"가 같은 무게.
  • 최고점/티어/온라인 상태가 카드에 묻혀 동기 부여 약함.
  • 설정·도움말이 시각적으로 버려진 위치.
재설계 요구
  • 위계: 주(빠른 한 판) → 모드 그룹(혼자/대결/순위) → 유틸. 모드 카드에 상태 메타(최고점·내 티어·온라인 인원/지연).
  • 로고/워드마크를 비주얼 의뢰서 브랜딩과 연결.
  • 하늘 순환과 호응하는 정적/미세모션 헤더.

상태: 최고점 0(숨김) vs 표시 · 온라인 가용/불가 · 진행 중 매치 "이어하기"(신규) · 버튼 hover/press/focus/disabled.

모션 최초 진입 카드 stagger rise, 모드 카드 hover lift, 하늘과 동기화된 헤더 톤.

5.2 게임 보드 + HUD (솔로 · 증강) GameScreen.tsx · Hud.tsx

현재

전체 화면 보드 호스트(Pixi) 위에 상단 HUD 한 줄. 점수=총점+라운드점수 연속 표기. 시간 바는 남은 비율, 남은 초가 ≤5면 .low 경고. 콤보>1일 때만 콤보 칩. 일시정지 버튼은 라운드 중에만 활성.

문제
  • 증강 모드인데 보유 빌드가 화면에 없음(사이드바는 대결 전용).
  • 점수가 "연속"이라 한 번의 큰 제거가 시각적으로 안 터짐.
  • 저시간 경고가 바 색만 — 긴박함 부족.
재설계 요구
  • 솔로/증강에도 빌드 칩 노출(§04.2 슬롯).
  • 큰 제거 시 점수 카운트 점프 + 보드 점수 팝업 동기화.
  • 저시간(≤5s, ≤3s) 단계적 긴박 연출(바+숫자+화면 가장자리 미세).

상태: 일반 · 잔여 ≤5초 · ≤3초 · 콤보 표시(2,3,…) · 드래그 중(진행/유효 박스) · 제거 직후 · 일시정지 진입 직전(라운드 phase에서만 일시정지 가능).

모션 점수 증가 카운트업( 처럼) · 콤보 칩 pop · 시간 바 저시간 펄스 · 일시정지 진입 시 보드 dim+blur 약하게.

5.3 대결 HUD (vs AI / 온라인) VersusHud.tsx · OnlineHud.tsx

현재

좌(나: 점수·콤보) · 중앙(R n/5·승수 me:opp·시간) · 우(상대: 이름·점수). AI 모드는 상대 득점 시 우측에 "+N" 플로팅이 떠오르고(score-pop), 코너에 AI 미니 보드(PiP)가 상대 수를 비춤. 온라인은 +N·미니뷰 없음, 상대 미입장 시 "(대기)"·시간 "—".

문제
  • 같은 "대결 HUD"가 두 컴포넌트로 갈려 피드백 비대칭(+N·미니뷰 유무).
  • 점수 역전 순간이 강조되지 않음 — 경쟁의 핵심 감정.
  • 승수(라운드 스코어보드)가 작은 텍스트라 시리즈 긴장 약함.
재설계 요구
  • +N·미니뷰·역전 강조를 공통화(온라인에도).
  • 역전/동점/리드 상태의 색·모션 신호.
  • 승수를 "베스트오브5" 점등 인디케이터로(라운드별 점 5개).

상태: 리드 / 동점 / 추격 / 역전 순간 · 잔여 ≤5초 · 콤보 · 상대 미입장(온라인) · 상대 +N · 미니뷰 on/off(설정).

모션 상대 +N floating(아래 +8) · 역전 시 양측 점수 스왑 강조 · 미니뷰 상대 마지막 수 0.5초 플래시(현재 구현됨).

5.4 증강 선택 AugmentOverlay.tsx (+ 온라인 인라인본)

현재

오버레이 패널: 티어 배지(실버/골드/프리즘) → 제목 "라운드 N · 증강 선택" → "하나를 골라 빌드를 쌓으세요 · 리롤 없음" → (대결만) 12초 카운트다운 바+숫자, 0이면 자동 선택 → 카드 3장(계열 아이콘+라벨 · 이름 · 효과 설명). 계열 아이콘은 이모지(⏱️🔥🍎📏🎲⚔️) 임시.

문제
  • 블라인드 픽: 지금 내 빌드를 못 본 채 선택(시너지 판단 불가).
  • 온라인 인라인본은 아이콘·카운트다운이 빠진 다른 패널.
  • 티어 위계가 배지 한 개 — "프리즘은 압도적"이 안 느껴짐.
  • "리롤 없음"의 긴장(되돌릴 수 없는 선택)이 연출로 없음.
재설계 요구
  • 패널에 현재 빌드 스트립 동반(획득 시 합류 애니메이션).
  • 단일 컴포넌트로 통합(온라인 포함), 카운트다운 항상.
  • 티어별 카드 프레임·광채·이펙트 위계(실버→골드→프리즘 점증).
  • 계열 6종 아이콘 정식화(이모지 대체) — 비주얼 의뢰서 아이콘 키트와 연동.

상태: 티어 3종 · 카드 hover/press/focus · 선택 직후(나머지 2장 dismiss) · 카운트다운 ≤3초 경고 · 자동 선택 발화 · (대결) 상대가 고르는 중 표시(선택적).

모션 증강 선택 시퀀스

카드 입장 stagger rise → 선택 카드 확대+발광, 미선택 2장 빠르게 후퇴 → 선택 증강이 빌드 스트립으로 날아가 합류(획득감) → 카운트다운 바는 시간 증강과 동일 이징. 프리즘은 입장 시 prism 그라데이션 스윕.

5.5 라운드 정산 · "라운드 점검" RoundCheckOverlay.tsx  핵심

현재 (대결 전용)

라운드 종료 후 3.5초 오버레이: 라운드칩 "라운드 N/총" → "라운드 점검" → 판정("내가 이긴 라운드!/상대가 이긴 라운드/비긴 라운드") → 나 점수 vs 상대 점수(이긴 쪽 .won 강조) → "누적 전적 me : opp" → 줄어드는 시간 바 → 자동으로 다음(증강)으로. ROUND_CHECK_MS=3500.

문제
  • 솔로/증강 모드엔 이 화면이 없음 — 라운드를 끝낸 보람 0.
  • 점수가 어떻게 합산됐는지 안 보임: 라운드 점수→누적 총점 반영, 라운드 승자 보너스(확정 스펙)가 화면에 전혀 드러나지 않음.
  • 3.5초 자동 진행만 — 스킵/머무름 불가, 정보 흡수 강제.
  • 누적 전적이 숫자 텍스트라 "베스트오브5"의 시리즈감 약함.
재설계 요구
  • 모든 모드 공통 라운드 정산(솔로=개인 베스트/델타 중심, 대결=대결 중심).
  • 정산 회계를 시각화: 라운드 점수 → (승자 보너스 +) → 누적 총점 갱신을 단계적 카운트업으로.
  • 스킵 가능(탭/스와이프) + 자동 진행 병행. 감속 선호 시 즉시.
  • 시리즈 인디케이터(라운드별 승패 점등).

상태: 내가 이긴 라운드 / 상대가 이긴 / 비김 · 시리즈 리드/추격/매치포인트(다음 라운드가 승부 확정 가능) · 마지막 라운드 직전(→ 결과로) · 솔로(개인 라운드 요약) 변형.

모션 라운드 정산 코레오그래피 (제안 타임라인 ~2.6–3.5s, 스킵 가능)

① 라운드 점수 양측 카운트업() → ② 승자 측 보너스 칩 pop(+보너스) → ③ 누적 총점 바/숫자 갱신(밀어 올림) → ④ 시리즈 점등 1칸 추가() → ⑤ "다음" 카운트다운 바. 매치포인트면 ④에서 강조 펄스. 솔로면 ①이 "이번 라운드 +N", ③이 개인 누적/신기록 추적 바.

5.6 매치 정산 · 결과 ResultOverlay · VersusResult · OnlineScreen  핵심

현재

솔로: "최고 기록!/게임 종료" → 큰 점수 + "점" → "최고 N · N라운드" → 다시 하기/홈으로.  대결: "승리!/패배/무승부"(.win/.loss/.draw) → 나 점수 : 상대 점수 → "라운드 me:opp" + (언랭크면)"연습(언랭크)" → 랭크면 "MMR ±N" → 다시 대결/홈으로.  온라인: 거의 동일 + 상대 이탈 시 "상대가 나갔습니다 · 부전승".

문제
  • 전부 정적 카드 — 카운트업/바/티어 변화 등 매치를 마무리하는 연출 0.
  • "MMR +N"이 숫자 한 줄 — 티어 게이지/승급·강등 순간이 없음(7티어 시스템인데).
  • 라운드별 흐름(어디서 이기고 졌나) 요약 없음.
  • 신기록이 텍스트 라벨뿐 — 자랑/공유 욕구 유발 약함.
  • 3벌 마크업이 갈려 연출을 한 번 만들어도 공유 불가.
재설계 요구
  • 단일 결과/정산 컴포넌트(솔로/대결/온라인 변형).
  • 연출: 최종 점수 카운트업 → 라운드별 스트립(R1–R5 결과) → (대결)MMR 바 채움 + 티어 칩 변화/승급 연출 → CTA.
  • 신기록 전용 강조(배지·파티클·"공유" 진입점, 비주얼 의뢰서 OG 카드 연계).
  • 부전승·무승부·언랭크 등 변형 명시.

상태: 솔로(신기록 / 일반) · 대결(승/패/무 × 랭크/언랭크) · 온라인(+ 상대 이탈 부전승) · MMR 상승/하락/승급/강등.

모션 매치 정산 시퀀스 (제안)

승/패/무 헤더 인 → 최종 점수 카운트업(양측 동시) → 라운드 스트립 좌→우 점등 → MMR 바 현재→신규로 채움() + 델타 칩 floating → 티어 경계 넘으면 승급 연출(티어 칩 교체+파티클) → CTA 등장. 신기록이면 점수 위로 신기록 배지 pop + 가장자리 파티클.

  • 신규 토스트(성공/정보/에러, 자동 소멸, 세이프영역 존중) · 스플래시/로딩(하늘 위 로고, 폰트/에셋 로드) · 표준 빈/에러 패턴 · 인앱 배너 재디자인.

문제 구조적 결함

① 그룹 없음: 솔로 관련(시간·개수·크기·사운드)과 대결 전용(AI 난이도·AI 미니화면)이 한 리스트에 뒤섞임.
② "다음 게임부터": 일시정지에서 바꿔도 지금 판에 반영 안 됨 → 혼란.
③ 미리보기 부재: "사과 개수/크기"를 바꿔도 효과를 다음 판까지 못 봄.
④ "개수"와 "크기"가 헷갈림: 개수=격자 차원, 크기=보드 전체 축소율(셀이 아니라 보드를 줄임) — 라벨만으론 모델이 안 읽힘.
⑤ 저장 피드백 없음 · 모드별 기본/프리셋 없음 · 접근성(세그먼트=라디오 그룹) 미흡.

재설계 요구 재설계

  • 카테고리화: 게임플레이(시간·개수·크기) / 대결(AI 난이도·미니화면) / 사운드·햅틱 / 접근성(모션 감소·고대비·숫자 크기) / 데이터(기본값·기록 초기화). 접근성·햅틱·모션감소는 신규 권장.
  • 라이브 미리보기: "사과 개수/크기"에 미니 보드 프리뷰(격자·셀이 실시간 변형). "개수↔크기"의 관계를 한 컨트롤로 직관화(2D 매트릭스 또는 프리뷰 동반 슬라이더).
  • 적용 모델 정리: "즉시 적용"을 기본으로, 라운드 중 바꿀 수 없는 것은 명확히 비활성/예약 표기. "다음 게임부터" 문구 의존 줄이기.
  • 저장/리셋 피드백(토스트), 기본값 복원 확인, 모드별 추천 프리셋(예: "모바일 최적", "스피드런").
  • 토글·세그먼트의 상태·모션·접근성(role=switch / radiogroup, 키보드, focus ring) 정식 컴포넌트화.

07라운드/매치 정산(Settlement) 심화  핵심

이 의뢰의 심장. 현재 "회계"는 코드 안에서만 일어나고 플레이어 눈앞에서 일어나지 않습니다.

점수 모델 (확정 스펙) 현재

라운드 점수제거할 때마다 가산. 콤보·증강이 곱/가산으로 개입(예: 연쇄 1.5×, 막판 스퍼트 2×, 도박사 ×3/×0.4).
매치 점수라운드 점수 누적 합산 + 라운드 승자 보너스 → 최종 누적 1위 승.
라운드 정산 데이터RoundResult{my, opp, winner, round} + 시리즈 roundWins{me,opp} + 누적 myTotal/oppTotal(versusStore).
타이밍 상수라운드 정산 ROUND_CHECK_MS=3500 · 증강 자동선택 AUGMENT_MS=12000 · (솔로엔 정산 단계 없음).
랭크 정산매치 후 ELO 적용 → mmrDelta. 봇 대결은 언랭크(MMR 미반영). 7티어 산출.

문제 정산이 "보이지 않는다"

① 라운드 점수가 어떻게 총점이 됐는지, 승자 보너스가 얼마였는지 화면에 없음(정산 오버레이는 라운드 점수와 전적 텍스트만).
② 솔로/증강은 라운드 정산 단계가 통째로 없음 → 5라운드를 쌓는 빌드의 성장이 "정산 모먼트"로 환산되지 않음.
③ 매치 결과의 MMR이 숫자 한 줄 → 티어/승급 서사 없음.
④ 정산이 전부 자동·정적 → 카타르시스 0.

재설계: 정산을 "감정의 회계장부"로 재설계

라운드 정산 (모든 모드)

① 이번 라운드 획득(콤보/증강 보너스 분해 표기 선택) → ② (대결)승자 보너스 → ③ 누적 총점 갱신 카운트업 → ④ (대결)시리즈 점등 / (솔로)개인 누적·신기록 추적 → ⑤ 다음으로(스킵 가능). 솔로는 가볍게, 대결은 경쟁 강조.

매치 정산 (단일 컴포넌트)

최종 점수 카운트업 → 라운드별 스트립(R1–R5 승패/점수) → (대결)MMR 바 채움+티어 변화+승급/강등 → 신기록/공유 → CTA. 부전승·무승부·언랭크 변형.

재설계/강화 요청 재설계

유효 박스의 "지금 놓으면!" 신호 강화, 콤보 단계별 버스트 누적 강화(2·3·5+), 실패 드래그의 절제된 거절 피드백, 특수 사과 식별성↔가독성 균형, 빈 칸 처리. 숫자 가독성(셀 ~20–50px)은 절대 최우선.

모션 스펙 납품 모션

파티클 수·속도·중력·수명, 팝업 드리프트/스케일, 박스 글로우 등을 수치 파라미터로(현재 값은 시작점). reduced-motion 대체. Pixi 포팅 메모는 apple_design_handoff/README 참고.

견제 FX (예정, 미리 시스템에 반영)

암전(화면 가림)·셔플(보드 재배치)·동결(입력 정지)·강탈(점수 흡수)·눈속임(숫자 흐림)·잡티(방해 사과). 상대를 흔드는 전체 화면 FX 레이어 + 내 화면이 당했을 때/상대에게 먹였을 때의 양방향 피드백을 모션 언어에 포함.

10반응형 · 입력 · 접근성 · 성능 예산

  • 입력: 마우스 드래그 + 터치 드래그(둘 다 1급). 텍스트 선택/탭 하이라이트 off, 전체화면 토글, 가로 모드 힌트. 정산 스킵·오버레이 dismiss의 제스처 정의.
  • 접근성: prefers-reduced-motion 완전 대응(정산·배경·이펙트), 포커스 가시화, 모달 포커스 트랩, 색 외 신호(승/패·유효), 숫자 대비, 세그먼트=radiogroup·토글=switch.
  • 성능 예산: 드래그 미리보기·파티클 60fps. 모바일 GPU 의식(과한 blur/filter·대형 그림자 남발 자제). 정산/전환이 입력 지연 유발 금지.
  • 폰트: 한글+숫자 가독, 웹폰트 서브셋/로딩 전략(스플래시와 연계).

12요청 산출물 & 핸드오프 (구현 가능 형태)

디자이너가 "구조 재설계 + 디자인 + 모션 구현"을 맡으므로, 개발이 바로 이식할 수 있는 형태를 요청합니다.

1) 구조/IA

  • 화면·상태 맵(전 모드)
  • 플로우/내비 다이어그램
  • 오버레이/z-레이어 규약
  • HUD 슬롯 시스템

2) 컴포넌트 스펙(전 상태)

  • 버튼/토글/세그먼트/인풋
  • 칩·배지(라운드/콤보/티어)
  • 카드(증강/정산/결과)
  • 토스트·스피너·빈/에러

3) 화면(모바일+데스크톱)

  • §05 전 화면·전 상태
  • 설정(§06) 재구조
  • 정산(§07) 전 변형
  • 로딩/스플래시

4) 모션

  • §09 마스터 표 확정 수치
  • 정산/카운트다운/MMR 시퀀스
  • Lottie 또는 CSS @keyframes
  • 보드 파티클 파라미터

5) 보드 에셋(Pixi)

  • 사과 normal/특수 스펙
  • 선택 박스(진행/유효)
  • 제거·콤보 이펙트
  • 견제 FX(예정 반영)

6) 토큰/시스템

  • 단일 토큰 소스(JSON)
  • CSS 변수 + theme.ts 파생
  • 타이포/간격/반경/그림자
  • 모션 토큰(듀레이션·이징)

납품 포맷(권장)

① Figma(오토레이아웃·컴포넌트·배리언트, 단일 진실 소스) · ② 토큰 JSON(Style-Dictionary 또는 CSS 변수+theme 숫자) · ③ 모션은 Lottie/짧은 영상 + 수치 스펙(§09) · ④ 아이콘/배지 SVG(+래스터 @1/2/3x) · ⑤ 레드라인/상태 스펙. React(CSS)+PixiJS로 바로 옮길 수 있으면 형식 자유.

14진행 순서 & 협업

제안 단계

  1. 킥오프 + 구조 합의 — IA/플로우 재설계(§03)·오버레이/HUD 시스템(§04)·정산 모델(§07) 방향 확정. 비주얼 의뢰서와 토큰 정합.
  2. 핵심 모먼트 먼저 — 게임 보드 HUD · 라운드/매치 정산 · 증강 선택을 우선 디자인+모션(이 게임의 감정선).
  3. 설정 + 로비 + 리더보드 + 시스템 — §06 재구조, 온라인 stage 통일, 토스트/로딩/에러.
  4. 모션 확정 + 핸드오프 — §09 수치 확정, Lottie/keyframes, 토큰 JSON, 보드 파라미터 납품.

킥오프 때 정할 것

  • 정산을 어디까지 화려하게(스킵 기본값·평균 체감 시간), 솔로 정산의 무게.
  • 온라인을 공통 매치 셸로 흡수하는 범위(구조 변경 폭).
  • 설정 "즉시 적용" 전환 범위(라운드 중 잠금 항목).
  • 견제 FX(예정)를 이번 모션 언어에 어디까지 미리 반영.
