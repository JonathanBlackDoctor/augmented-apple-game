## 0. 한눈에 (TL;DR)

- **무엇을** 22개 증강 각각의 **아트 엠블럼**(아이콘/일러스트)을 1종씩.
- **어떤 스타일로** 첨부 레퍼런스 `증강카드_시안/시안_채색형_정교.png`와 **동일한 톤**(따뜻한 과수원 팔레트 + 광택 일러스트).
- **어떻게 납품** 각 엠블럼을 **SVG**(`viewBox="0 0 100 100"`) + **PNG @1x/2x/3x**, 지정된 **파일명 규칙**으로.
- **합격 기준** 6계열 시각 정체성이 구분되고, 3티어 위계가 보이며, 16px까지 줄여도 실루엣이 읽힐 것.

> 우리가 직접 만든 **샘플 4종**(황금 사과·유리대포·잔상·재배치)이 첨부되어 있습니다. **이것이 목표 품질·톤의 기준선**입니다. 같은 결을 유지하되, 더 정교하고 아름답게 끌어올려 주세요.

## 1. 게임 컨텍스트 (왜 이 아트가 필요한가)

"합이 10이 되도록 숫자 사과를 드래그로 담는" 사과게임 코어 위에, **리그 오브 레전드 아레나식 증강 빌드**와 실시간 1대1 대결을 얹은 웹 게임입니다.

라운드마다 플레이어는 **증강 카드 3장 중 1장**을 골라 빌드를 쌓습니다(리롤 없음). 이 **증강 선택 순간**이 게임의 핵심 쾌감 지점이며, 카드 아트는 그 선택을 **탐나고 설레게** 만드는 역할을 합니다.

**경험 우선순위** ① 즉각적 손맛 › ② **빌드의 쾌감(증강)** › ③ 공정한 긴장감 › ④ 바이럴 성장. 카드 아트는 ②를 직접 책임집니다.

## 3. 스타일 디렉션 (Art Direction)

### 3.1 톤 & 무드 — "햇살 과수원 (Sunlit Orchard)"
잘 익은 과수원의 따뜻한 햇살. 크림빛 종이 위에서 윤기 흐르는 사과가 톡톡 터지는 — **정성 들인 프리미엄 과일 브랜드** 같은 라이트·캐주얼 톤. 어둡거나 차갑지 않게, 항상 **따뜻하고 먹음직스럽게**.

### 3.2 표현 양식
- **채색형(면 기반) 일러스트.** 단순 라인 아이콘 ❌. 입체감 있는 채색 엠블럼 ⭕.
- **다층 그라데이션** — 형태마다 2~3 스톱 이상으로 양감 표현.
- **스페큘러 하이라이트** — 윤기나는 광택점(샤프) + 부드러운 시인(sheen).
- **소프트 그림자/글로우** — 접지 그림자, 광원 헤일로, 미세 글로우로 공간감.
- **사과 실루엣 모티프** — 게임의 핵심 오브젝트. 가능하면 엠블럼에 사과의 통통한 실루엣 DNA를 녹여 주세요(아래 §4.3 path 제공).
- **메달형 수치 배지** — 점수 배수 등 수치(×2, ×3, 10)는 작은 원형 메달로 표현(샘플 참조).

### 3.3 레퍼런스
- ✅ **목표 기준선**: `증강카드_시안/시안_채색형_정교.png` (우리 제작 샘플 4종)
- ✅ **기존 3종 톤 참조**: `moodboard.jsx` / `styles.css`의 `ART` 객체(time·combo·rule 엠블럼)
- ✅ **생성 코드 참조**: `증강카드_시안/build-cards.mjs` — 샘플 4종을 만든 SVG 코드. 그라데이션·필터·메달·사과 글리프 헬퍼가 그대로 들어있어, 같은 방식으로 확장하면 됩니다(참고용이며 강제 아님).

## 4. 카드 슬롯 규격 (엠블럼이 놓이는 환경)

엠블럼은 **카드 상단 아트 영역**에 들어갑니다. 디자이너는 엠블럼만 그리면 되지만, 아래 환경에서 보기 좋도록 맞춰 주세요.

### 4.1 카드 & 아트 영역 치수 (현재 구현 기준)
| 요소 | 값 |
|---|---|
| 카드 | 244 × 338 px, radius 20 |
| 아트 영역 | 카드 폭 - 34, 높이 **172 px**, radius 15 |
| 엠블럼 캔버스 | **`viewBox="0 0 100 100"`**, 아트 영역 중앙 배치(약 1.4~1.5× 스케일로 표시됨) |
| 안전 여백 | 100 단위 캔버스 기준 **상하좌우 8 이상** 여백 유지 (글로우/그림자는 넘어가도 됨) |

## 5. 계열 & 티어 시각 시스템

### 5.1 6개 계열 — 각각 고유한 시각 정체성
각 계열은 **모티프·색 힌트**로 한눈에 구분되어야 합니다.

| 계열 | 키워드 | 색/모티프 힌트 |
|---|---|---|
| **시간 (time)** | 여유·감속·정지 | 시계/모래시계/잔상 트레일, 은빛·금빛 |
| **콤보 (combo)** | 연속·배수·폭발 | 별/스파크/연쇄 화살, 골드·옐로 |
| **보드 (board)** | 판을 유리하게 | 사과 그 자체(황금/와일드)·격자·정리, 따뜻한 레드/골드 |

| **견제 (disrupt)** | 상대 방해(대결) | 흐림·셔플·동결·암전, 차갑거나 불길한 악센트 |

## 7. 기술 납품 사양 (Handoff Spec)

### 7.1 포맷
- **SVG** (벡터 원본) — `viewBox="0 0 100 100"`, 그룹/레이어 정리, 텍스트는 가능하면 path 변환(웹폰트 의존 제거). 그라데이션/필터 id는 **엠블럼별 고유 접두사**로 충돌 방지.
- **PNG 래스터** — 투명 배경, **@1x(100px) / @2x(200px) / @3x(300px)** 3종.
- (선택) Figma 원본 또는 컴포넌트 — 있으면 가산점.

### 7.2 파일 네이밍 규칙 (엄수)
```
augment-<id>.svg            예) augment-board.golden.svg
augment-<id>@1x.png / @2x.png / @3x.png
```
- `<id>`는 §6 표의 ID를 그대로 사용(점 포함). 예: `time.afterimage`, `disrupt.blackout`.

### 7.3 폴더 구조 (제안)
```
augment-cards/
  svg/        augment-time.relief.svg ...
  png/        augment-time.relief@1x.png ...
  _preview/   계열별/티어별 시트(전체 한눈에)
  README       계열·티어 매핑, 색 사용 노트
```

### 7.4 색/렌더 제약
- 모든 색은 §4.2 토큰을 기본으로, 계열 정체성 색을 악센트로. **임의의 새 베이스 컬러 도입은 사전 합의.**
- 그림자/글로우는 SVG 필터(`feGaussianBlur`/`feDropShadow`) 또는 면으로 표현 — **@1x 100px, 16px 썸네일에서도 깨지지 않게** 디테일 위계 관리.
