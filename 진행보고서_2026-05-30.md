## 1. 이번 세션에 완료한 것

### Wave 0 — 스캐폴드 · 계약 · 툴링
- Vite 5 + React 18 + TypeScript(strict) 스캐폴드, 단일 앱 + 모듈 경계 구조(계획서 §3 디렉터리 그대로).
- **`src/contracts/*` 전체 확정** (계획서 §5): `core / rng / clock / augment / roundmode / net / profile / matchmaking` + 배럴 `index.ts`. 로직 0, 타입·인터페이스·이벤트만.
- **ESLint 코어 순수성 규칙 강제**(계획서 §3): `src/core/**` 에서 `react·react-dom·pixi.js·firebase·zustand` import 금지, DOM 전역/`Date`/`performance`/`requestAnimationFrame` 금지, `Math.random`·`Date.now` 금지. → 위반 파일로 **실제 차단됨을 확인**(react import·Math.random·Date.now·window 모두 error).
- Prettier, Vitest(노드 환경), GitHub Actions 배포 워크플로(`.github/workflows/deploy.yml`), `public/404.html`(SPA 폴백)·`manifest.webmanifest`·`favicon.svg`.
- Firebase 보안 규칙 초안(`firebase/database.rules.json`, 계획서 §6) + 에뮬레이터 설정(`firebase/firebase.json`).

### Phase 0 — 코어 싱글 플레이 (혼자 30초 사과게임)
- **순수 코어 엔진**(`src/core`): 시드 기반 보드 생성, 합10 판정(직사각형 선택·빈칸 제외), 점수(사과 1개=1점), 주입 단조시계 기반 타이머, **결정적 `replay()`**(재시뮬레이션). `mulberry32` 시드 RNG(`fork` 동기화 지원).
- **보드 렌더**(`src/board`): PixiJS v8 — 사과 격자·번호·선택 박스(유효 시 초록)·제거 파티클·"미니멀 프리미엄" 디자인 토큰(계획서 부록 A). 순수 기하(`layout.ts`)는 별도 단위테스트.
- **입력**(`src/input`): 포인터/터치 드래그 → 픽셀→격자 변환 → 선택 사각형 → core 커밋. 캔버스 `touch-action:none`로 모바일 스크롤 방지.
- **런타임 지휘자**(`src/app`): `MatchController`(엔진·보드·입력·시계·사운드·스토어 와이어링), zustand 스토어, `performance.now` 래퍼 단조시계, WebAudio 효과음(손맛).
- **UI**(`src/ui`): 홈 / 게임 HUD(점수·시간바·콤보) / 결과 화면. 최고점 `localStorage` 영속.

### Phase 1 — 증강 루프 + 각자 보드 + 5라운드 진행
- **증강 시스템**(`src/augments`): 훅 버스(7개 훅 지점 폴딩), 데이터 주도 카탈로그 **12종**(시간·콤보·보드·룰·하이리스크 계열), 티어 스케줄(R1·R2 실버 → R3·R4 골드 → R5 프리즘), **결정적 3택1 롤**(리롤 없음, 보유/충돌 제외).
- **대결 모드**(`src/roundmodes`): `separate`(각자 동일 시드 보드) `buildRound`/`scoreRound` 구현.
- **5라운드 매치 루프**: 라운드마다 증강 3택1 선택 → 라운드 플레이 → 누적 점수. 증강은 매치 내 누적되어 코어 동작을 패치(시간 연장/감속/정지, 콤보 배수, 황금 사과, 합9·11·10의배수 인정, 점수 3배 등).

## 2. 검증 결과 (실제 실행 출력)

| 항목 | 결과 |
|---|---|
| 단위 테스트 (`npm test`) | **36개 전부 통과** — core 14, augments 9, rng 6, layout 4, roundmode 3 |
| 타입체크 (`tsc --noEmit`) | 오류 0 |
| 프로덕션 빌드 (`npm run build`) | **성공** — 780 모듈, 메인 번들 435KB(gzip 139KB), Pixi 자동 코드분할 |
| ESLint (`npm run lint`) | 통과(0 error) + 코어 순수성 규칙 위반 차단 확인 |

핵심 코어 테스트(계획서 §18 최소 기준)를 모두 충족:
- 합10 판정: 경계(미만/초과)·빈칸 제외·다중 사과(2D)·역방향/범위초과 정규화.
- 점수: 제거 사과 수 = 점수, 무효 선택 거부 시 점수 불변.
- **결정성**: 동일 시드 → 동일 보드(엔진·generateBoard 양쪽), 다른 시드 → 다른 보드.
- **재시뮬레이션 일치**: 동일 action 로그 → 동일 점수(같은 엔진·새 엔진 모두). 증강 포함 replay 일치도 검증. 조작된 로그(가짜 클리어)는 점수 미반영.
- 타이머: 정확히 0에서 종료, 종료 후 커밋 거부.

> 주의: 빌드 환경에 디스플레이가 없어 **Pixi 보드의 시각적 렌더는 육안 확인 불가**입니다. 플레이 가능 여부는 빌드 성공 + 코어/기하 단위테스트로 보증했습니다. 사용자가 `npm run dev`로 손맛(드래그·파티클·사운드)을 최종 확인해 주세요.

## 3. 로컬 실행 방법

## 4. 막힌 지점 & "사용자가 직접 해야 할 일" (외부 계정·자격증명 필요)

아래는 자격증명이 필요해 에이전트가 수행할 수 없는 작업입니다. 코드/규칙 초안은 이미 들어가 있으니, 키만 넣고 연결하면 됩니다(계획서 §19).

### A. Firebase (실시간 백엔드 — Phase 2부터 필요)
1. [Firebase 콘솔](https://console.firebase.google.com)에서 프로젝트 생성.
2. **Realtime Database** 활성화(리전 선택) + **Authentication → 익명(Anonymous)** 로그인 사용 설정.
3. 웹 앱 등록 후 config 객체를 `src/net/firebaseConfig.ts` 의 placeholder에 붙여넣고, 같은 파일의 `FIREBASE_CONFIGURED`를 `true`로.
4. 보안 규칙 배포: 콘솔에 `firebase/database.rules.json` 내용을 붙여넣거나 `firebase deploy --only database`.
5. (배포 후) GitHub Pages 도메인을 Auth **승인된 도메인**에 추가. (선택) App Check(reCAPTCHA v3).

### B. GitHub 저장소 + Pages 배포
1. 이 폴더를 깃 저장소로 만들어 GitHub에 푸시(`main` 브랜치). `node_modules`·`dist`는 `.gitignore`로 제외됨.
2. 저장소 **Settings → Pages → Build and deployment → Source = GitHub Actions**.
3. 푸시하면 `.github/workflows/deploy.yml`이 `npm ci → lint → test → build → Pages 배포`를 자동 수행.
4. **base 경로**: 현재 `vite.config.ts`의 `base: './'`(상대 경로)로 프로젝트 페이지 하위경로·로컬 미리보기 모두 동작. 절대경로를 원하면 `'/<레포이름>/'`으로 변경.

### C. Kakao 공유 (Phase 5)
1. [developers.kakao.com](https://developers.kakao.com)에서 앱 생성 → **JavaScript 키** 발급.
2. 플랫폼(Web)에 Pages 도메인 등록.
3. `Kakao.init(JS_KEY)` 초기화(키 공개 전제). 공유는 도메인 등록만으로 동작. → `src/social` 구현 시 사용(미구현).

> 참고: 작업 폴더에 0바이트 `_t` 파일이 하나 있습니다(샌드박스 디스크 점검 잔여물, 마운트 권한상 삭제 불가). 무해하니 직접 지우셔도 됩니다.

## 6. 이번 세션의 설계 결정·계약 보완 (계획서 대비 편차)

오케스트레이터 권한으로 계약을 다음과 같이 확정·보완했습니다(모두 계획서 의도 보존):
- `makeRng` 구현은 `src/core/rng.ts`에 둠(계약 `rng.ts`는 인터페이스만 — "로직 0" 유지).
- `Board`에 선택적 `tags`(normal/golden/wild) 계층 추가 — 황금/와일드 사과를 결정적으로 표현.
- `ClearCtx`에 `clearedValues·clearedTags·grantTimeMs` 추가, `CoreEngine`에 `setDragging` 추가 — 시간/황금 계열 증강을 기존 훅으로 깔끔히 구현.
- 보드/증강 RNG는 **`cfg.seed`에서 파생**(주입 rng와 무관) → `replay` 완전 결정성. 주입 rng는 Phase 4 견제 동기화용으로 예약.
- Phase 1 증강은 **단독 플레이에 유효한 12종 큐레이션**: `disrupt`(견제) 계열은 상대가 필요하므로 솔로 제안에서 제외, `친절`은 결정성을 위해 확률 제거(합9 항상 인정)로 단순화. 나머지 카탈로그는 Phase 4~5에서 확장.
- `vite.config.ts base: './'` 채택(레포명 하드코딩 회피, Pages 하위경로+로컬 동시 호환).
