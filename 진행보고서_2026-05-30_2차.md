## 1. 이번 세션에 추가한 것

### 지금 바로 플레이 가능 — **AI 대결 모드 (오프라인, 백엔드 불필요)**
홈 화면에 **"AI와 대결 · 5라운드"** 버튼이 생겼습니다. 5라운드 동안 AI 봇과 점수를 겨루고, 라운드 사이 증강을 3택1로 골라 빌드를 쌓습니다. 라운드 승자 보너스·누적 점수·최종 승패가 계산되고, 내 MMR에 맞춰 봇 난이도가 자동 조절됩니다(봇전은 언랭크).

## 4. 아직 남은 것 (정직한 현황)

이번 세션은 **오프라인 대결까지** 완성했고, **실시간 온라인 1대1**은 *부품은 다 만들어졌지만 끝단 통합이 남았습니다*.

- **온라인 사람 vs 사람**: `VersusMatch`가 현재 상대로 *봇*을 구동합니다. 이를 *원격 상대(net `clear` 이벤트로 상대 점수 반영)*로 바꾸고, **온라인 매칭/방 입장 UI**를 추가하고, **Firebase 백엔드 어댑터**(인메모리 백엔드 자리에 RTDB 연결)를 붙이면 완성됩니다. 필요한 토대(NetSession·인메모리 백엔드·매치메이커·프로필·랭킹)는 **모두 구현·테스트 완료**.
- **Phase 4 라이브 견제**: 같은보드 클레임·결정적 셔플·견제 모드는 만들었지만, 엔진의 `emitSabotage/onIncomingSabotage` 훅 연결과 견제 계열 증강 6종의 라이브 적용은 남았습니다.
- **소셜 마무리**: 카카오 공유(키 필요)·초대 보상 루프·데일리 챌린지 UI 화면.

## 5. 사용자가 해야 할 일 — 외부 계정 설정 & 배포

코드/규칙 초안은 이미 들어가 있습니다. 아래는 **자격증명이 필요해 제가 대신 못 하는** 단계입니다. (원하시면 제가 브라우저로 함께 진행할 수 있어요 — §6.)

### A. 지금 바로 무료 배포 (GitHub Pages) — 백엔드 없이 솔로·증강·AI대결 공개
1. 이 폴더를 git 저장소로 만들어 GitHub에 푸시(`main`). `node_modules`·`dist`는 `.gitignore`로 제외됨.
2. 저장소 **Settings → Pages → Source = GitHub Actions**.
3. 푸시하면 `.github/workflows/deploy.yml`이 `lint → test → build → 배포`를 자동 수행 → `https://<사용자명>.github.io/<레포명>/` 로 공개.
   - 현재 `vite.config.ts`의 `base: './'`라 하위경로·로컬 모두 동작.

### B. Firebase (실시간 온라인 1대1 — 위 §4 통합 후 필요)
1. [Firebase 콘솔](https://console.firebase.google.com) → 프로젝트 생성.
2. **Realtime Database** 활성화 + **Authentication → 익명** 사용.
3. 웹 앱 등록 → config를 `src/net/firebaseConfig.ts`에 붙여넣고 `FIREBASE_CONFIGURED=true`.
4. `firebase/database.rules.json` 규칙 배포 + Pages 도메인을 Auth 승인 도메인에 추가.

### C. Kakao 공유 (선택)
1. [developers.kakao.com](https://developers.kakao.com) → 앱 생성 → **JavaScript 키**.
2. `src/social/share.ts`의 `kakaoConfig.jsKey`에 입력 + 도메인 등록.
