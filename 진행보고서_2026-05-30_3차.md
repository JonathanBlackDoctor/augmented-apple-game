- **일자**: 2026-05-30
- **이번 세션**: 온라인 실시간 1대1 구현 + Firebase 연동 + 견고화/리더보드 (사용자 개입 없이 가능한 범위 전부)
- **검증**: 타입체크·린트·프로덕션 빌드·테스트 **80개** 전부 녹색.

## 1. 지금 플레이 가능한 모드 (전부 빌드 검증됨)

1. **빠른 한 판** — 혼자 30초.
2. **5라운드 증강 모드** — 혼자, 증강 빌드.
3. **AI와 대결** — 봇과 5라운드(증강 포함). *오프라인·라이브 확인 완료.*
4. **친구와 1:1 대결 (온라인)** — 방 만들기/코드 입장/초대 링크(`?room=`) 자동 입장 → 5라운드 실시간, 상대 점수 실시간 + 증강 + 승패/ΔMMR. *코드 완성·검증, 라이브 2기기 테스트는 push 후.*
5. **랭킹** — 상위 20 리더보드(Firebase 영속).

## 2. 이번 세션에 추가한 것

### 온라인 실시간 1대1 (Firebase RTDB)
- Firebase 콘솔에 **프로젝트 생성 · RTDB · 익명 인증 · 웹앱 등록**까지 완료, config를 코드에 반영(`FIREBASE_CONFIGURED=true`). firebase SDK는 **지연 로딩 청크**(gzip 73KB)라 오프라인 플레이엔 안 받음.
- `OnlineMatch`: 시간 기반 스케줄(카운트다운 → 5×[라운드+증강])로 양 클라이언트 동기화, 상대 점수는 `clear` 이벤트로 추적, 증강 각자 픽. **인메모리 2-클라이언트 풀매치 시뮬레이션 테스트** 통과.
- `OnlineController` + 로비/HUD/결과 UI, 익명 프로필(RTDB), 랭크(ELO) 적용.

### 견고화 (이번에 보강)
- **상대 이탈 = 부전승**: 하트비트로 상대 생존 감지, 끊기면 일정 시간 뒤 자동 승리 처리. (테스트 포함)
- **미입장 타임아웃 → AI 전환**: 방에 아무도 안 오면 "AI와 대결로 전환" 버튼. (테스트 포함)
- **2인 정원 잠금**: 같은 방에 3번째가 들어와도 첫 상대에만 고정, 난입 이벤트 무시. (테스트 포함)
- **인앱 브라우저 배너**: 카톡/인스타 인앱으로 열면 "외부 브라우저로 열기" 배너 표시.
- **닉네임 입력 UI**: 온라인 로비에서 닉네임 변경.
- **리더보드 영속 + 화면**: 랭크 결과를 Firebase `/leaderboard`에 기록, 상위 20 조회 화면.

## 5. 남은 후속조치 (사용자/라이브/유료가 필요해 자율로 못 한 것)

### A. 사용자·라이브가 필요
- **라이브 2기기 테스트** — 실제 지연·순서에서 동기화 최종 확인(인메모리 시뮬만 통과). 제가 두 탭으로 같이 확인 가능.
- **RTDB 정식 규칙 적용** — 현재 테스트 모드(30일 개방). `firebase/database.rules.json`로 교체.
- **Auth 승인 도메인에 Pages 주소 추가**(익명 로그인은 보통 없어도 동작, 문제 시).

### B. 대형·리스크라 라이브 검증과 함께 해야 안전
- **견제(sabotage) 온라인 연결** — 셔플/암전/강탈/동결 + 견제 증강 6종. 부품(shuffleOrder·sabotage 모드·claim)은 만들어 테스트했고, 엔진 훅 연결 + 카탈로그 추가가 남음(코어 변경이라 플레이테스트 권장).
- **같은보드 쟁탈 모드 온라인 적용** — `resolveClaim`/트랜잭션 준비됨.
- **치팅 검증(replay)** — 상대 액션 로그를 `core.replay()`로 재시뮬 검증(§10.4).
- **봇·증강 밸런스** — 수치 튜닝(플레이 피드백 필요).

### C. 유료 플랜(Blaze) 필요
- **서버 권위 MMR / RTDB 자동 청소(Cloud Functions)** — 지금은 클라이언트 계산 + 방 데이터 누적.

### D. 기타 미구현(자율 가능하나 이번엔 보류)
- **카카오 공유**(JS 키 필요) · **초대 보상 루프** · **데일리 챌린지 화면** · **시계 정밀 동기화(serverTimeOffset)** · **Playwright E2E**.

> 우선순위만 주시면 B/D 항목을 이어서 진행합니다. 먼저 **push → 라이브 2기기 테스트**가 가장 큰 다음 한 걸음입니다.
